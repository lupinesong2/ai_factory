# 냉장고 재료 기반 레시피 생성 스킬

## 이 스킬이 하는 일

사용자의 냉장고 재료(`01_refrigerator/ingredients.json`)를 읽고, 그 재료로 **실제로 만들 수 있는** 한식 레시피 여러 개를 생성합니다. 그리고 결과를 **마크다운(.md) 파일**로 저장합니다.

하드코딩된 레시피 DB를 쓰지 않습니다. 재료 조합·수량·보관 상태(냉장/냉동)를 실제로 살펴보고, 매 실행마다 그 냉장고에 맞는 레시피를 새로 구성해야 합니다. 그게 이 스킬의 핵심 가치입니다.

## 워크플로우

### 1. 재료 읽기

`01_refrigerator/ingredients.json`을 Read 툴로 읽습니다. 경로는 현재 작업 디렉토리 기준입니다. 파일이 없으면 사용자에게 경로를 물어보세요.

JSON 구조 예시:
```json
{
  "refrigerator": {
    "lastUpdated": "2026-04-22",
    "ingredients": [
      { "name": "삼겹살", "category": "육류", "quantity": "500g", "storage": "냉동" },
      ...
    ]
  }
}
```

### 2. 레시피 생성

재료를 훑어본 뒤, 그 중 실제 활용 가능한 조합으로 **3~5개의 한식 레시피**를 만듭니다. 각 레시피는 다음을 만족해야 합니다.

- **실현 가능**: 냉장고에 있는 재료로 주요 재료가 커버되어야 함. 양념(소금·간장·참기름·식용유·설탕·마늘·후추 같은 기본 양념)은 "집에 있다고 가정"해도 OK. 하지만 주재료(고기·채소·유제품 등)는 반드시 냉장고에 있어야 함.
- **다양성**: 비슷한 요리만 3개 내놓지 마세요. 예: 찌개·볶음·구이·밥·국·반찬 등 **조리법이 겹치지 않도록** 구성하세요. 사용자가 한 상 차리는 기분이 들게.
- **재료 활용도**: 유통기한이 짧거나 수량이 많은 재료(예: 500g 삼겹살, 10개 계란)는 우선적으로 활용하세요. 냉동 재료는 해동 단계를 명시.
- **한국식 가정식**: 거창한 코스 요리보다는 실제 집에서 해먹는 메뉴로. 김치찌개·된장찌개·볶음밥·전·국·무침·구이 등.

각 레시피는 아래 정보를 포함해야 합니다.
- 이름
- 조리 시간 (예: "20분")
- 난이도 (쉬움/보통/어려움)
- 사용 재료 (냉장고에서 꺼낸 것 + 기본 양념, 각 수량 포함)
- 조리 순서 (4~7단계, 구체적으로)
- 팁 한 줄 (있으면 좋음 — 예: "삼겹살은 키친타월로 기름기 닦으면 더 담백해요")

### 3. 마크다운 파일 생성

결과를 마크다운(.md) 파일로 작성합니다. 저장 위치는 **재료 JSON 파일과 같은 폴더**(보통 `01_refrigerator/`)에 `recipes_YYYYMMDD_HHMM.md` 이름으로. 타임스탬프를 붙이는 이유는 매번 새로 생성되는 추천이 덮어쓰이지 않게 하기 위함입니다.

마크다운 문서 구조 (이 순서를 따르세요):

```markdown
# 🍲 오늘의 냉장고 레시피

> 생성: YYYY-MM-DD HH:MM · 냉장고 재료 N종 · 추천 레시피 M개

## 🥬 냉장고 재료

사용한 재료는 **굵게** 표시합니다.

- **삼겹살** (500g, 냉동)
- 닭가슴살 (300g, 냉동)
- **계란** (10개, 냉장)
- ...

---

## 1. 돼지고기 김치찌개

- ⏱ 조리 시간: 30분
- 🔥 난이도: 쉬움
- 👥 인분: 2~3인분

### 재료
- 삼겹살 250g
- 묵은 김치 1.5컵
- 두부 반 모
- ...

### 만드는 법
1. 첫 번째 단계 설명...
2. 두 번째 단계 설명...
3. ...

> 💡 **Tip.** 팁 한 줄

---

## 2. 다음 레시피...
```

작성 규칙:
- **이모지는 적당히**: 헤더(🍲🥬)와 메타정보(⏱🔥👥💡)에만. 본문은 깔끔하게.
- **재료 리스트**: 냉장고 재료 섹션에서 이 레시피들에 쓰이는 재료는 `**굵게**` 표시. 안 쓰인 재료도 다 보여주되 일반 텍스트로.
- **레시피 간 구분**: `---` 수평선으로 구분.
- **Tip은 blockquote** (`> 💡 **Tip.** ...`)로.
- UTF-8로 저장 (한글 깨짐 방지).

### 4. 완료 알림

파일 생성 후, 사용자에게 파일 경로를 명확히 알려주세요. 마크다운은 별도로 열지 않아도 됩니다 (VS Code·IDE에서 미리보기 가능하고, 텍스트 에디터로도 읽기 좋으니까).

### 5. 대화창에는 간단히 요약

긴 레시피 본문을 대화창에 반복해서 쓰지 마세요 (마크다운 파일에 다 들어있으니까). 대화창에는:
- 생성한 레시피 개수와 제목 리스트
- 마크다운 파일 경로
- 한두 줄 코멘트 (예: "삼겹살이 500g이나 있어서 김치찌개랑 삼겹살 양파볶음 두 개에 나눠 썼어요")

이렇게 짧게 마무리하세요.

## 왜 이렇게 설계했는가

- **레시피를 하드코딩 안 함**: 사용자의 냉장고는 매번 다릅니다. 삼겹살 500g에 시금치가 없을 때와, 닭가슴살에 시금치가 있을 때의 최적 메뉴는 달라져야 합니다. LLM이 매번 판단하는 게 이 스킬의 가치예요.
- **마크다운 출력**: 레시피는 보통 "보면서 따라하는" 문서라 채팅창 한 번 흘러가는 답변보다 파일로 남기는 게 유용합니다. 마크다운은 IDE·에디터·깃허브에서 바로 미리보기 되고, 수정·공유·버전 관리가 쉽고, 나중에 다른 포맷(HTML·PDF)으로 변환하기도 편함.
- **타임스탬프 파일명**: 같은 재료로 여러 번 실행해서 다른 추천을 비교해볼 수 있도록.
- **양념은 있다고 가정**: 한국 가정에는 소금·간장·고추장·된장·마늘 등이 있는 게 보통이고, 이걸 제약으로 걸면 레시피 후보가 너무 줄어듭니다. 단, JSON에 있는 양념(예: 고추장·된장)은 "확실히 있으니 적극 써도 됨"으로 취급.

## 엣지 케이스

- **재료가 너무 적음 (3개 미만)**: 2~3개 레시피만 만들되, "이 재료로는 선택지가 제한적이에요. ○○, ○○를 추가하면 △△도 가능해요" 같은 제안 한 줄 덧붙이기.
- **재료 카테고리 쏠림 (예: 전부 채소)**: 채식 메뉴 위주로 구성. 굳이 고기 들어가는 요리 만들려고 억지 부리지 말 것.
- **파일 경로를 찾을 수 없음**: 현재 디렉토리 + `01_refrigerator/ingredients.json` 순으로 시도하고, 그래도 없으면 사용자에게 경로를 물어보기.
- **수량이 명시되지 않은 재료**: 그냥 "적당량"으로 처리.
